## 一、架构设计

### 1.1 当前架构

```
┌─────────────────────────────────────────────────────────────┐
│                    MF Shell 基座 (Vue3)                      │
│                    端口: 3041                                │
└──────────────────────────┬──────────────────────────────────┘
                           │
         ┌─────────────────┴─────────────────┐
         ▼                                   ▼
┌──────────────────┐              ┌──────────────────┐
│ 权限中心前端      │              │ 用户中心前端      │
│ (React + AntD)   │              │ (Vue3 + TDesign) │
│ 端口: 3000        │              │ 端口: 3042        │
└────────┬─────────┘              └────────┬─────────┘
         │                                  │
         ▼                                  ▼
┌──────────────────┐              ┌──────────────────┐
│ 权限中心后端      │              │ 用户中心后端      │
│ (Spring Boot)    │              │ (NestJS)         │
│ 端口: 8080        │              │ 端口: 4120        │
└──────────────────┘              └──────────────────┘
```

**问题：**
1. 三个独立的进程，需要分别启动
2. 用户中心和权限中心各自有用户认证
3. 没有统一的权限控制
4. API 地址硬编码

### 1.2 目标架构

```
┌─────────────────────────────────────────────────────────────┐
│                  权限中心前端 (React + Ant Design)           │
│                  统一登录入口 + Tab 模块切换                  │
│                  端口: 3000                                 │
├─────────────────────────────────────────────────────────────┤
│  ┌─────────────┐  ┌─────────────┐  ┌─────────────────────┐ │
│  │ 用户中心    │  │ 权限管理    │  │   系统设置(未来)     │ │
│  │  (iframe)   │  │  (原生)     │  │                     │ │
│  └─────────────┘  └─────────────┘  └─────────────────────┘ │
└──────────────────────────┬──────────────────────────────────┘
                           │
         ┌─────────────────┴─────────────────┐
         ▼                                   ▼
┌──────────────────┐              ┌──────────────────┐
│ 权限中心后端      │              │ 用户中心后端      │
│ (Spring Boot)    │◄────────────►│ (Spring Boot)    │
│ 端口: 8080        │   统一鉴权    │ 新增用户管理API   │
│                  │              │ 端口: 8081        │
└──────────────────┘              └──────────────────┘
         │                                  │
         └────────────┬─────────────────────┘
                      ▼
              ┌──────────────────┐
              │   MySQL 数据库    │
              │   统一数据存储     │
              └──────────────────┘
```

**改造策略：**
1. 保留权限中心前端作为主入口
2. 用户中心通过 iframe 嵌入（技术栈隔离）
3. 新增 Spring Boot 用户管理模块（替代 NestJS）
4. 统一使用权限中心的认证和鉴权

```java
/**
 * 获取当前用户权限列表
 */
@GetMapping("/api/auth/permissions")
public ApiResponse<UserPermissionVO> getUserPermissions() {
    String userId = getCurrentUserId();

    // 1. 查询用户基本信息
    UserDO user = userService.getByUserId(userId);

    // 2. 查询用户权限列表
    Set<String> permissions = authzService.getUserPermissionCodes(userId, null);

    // 3. 解析可访问的模块
    Set<String> modules = parseModules(permissions);

    UserPermissionVO vo = UserPermissionVO.builder()
        .userId(userId)
        .userName(user.getDisplayName())
        .permissions(permissions)
        .modules(modules)
        .build();

    return ApiResponse.success(vo);
}

private Set<String> parseModules(Set<String> permissions) {
    Set<String> modules = new HashSet<>();
    for (String permission : permissions) {
        if (permission.startsWith("USER_CENTER")) {
            modules.add("user-center");
        } else if (permission.startsWith("PERMISSION_MANAGE")) {
            modules.add("permission-manage");
        }
    }
    // 所有用户默认可访问用户中心
    modules.add("user-center");
    return modules;
}

          }))}
          onClick={({ key }) => setActiveModule(key)}
        />
      </Sider>

          />
        ) : (
          <Outlet />  /* 权限管理模块使用原生路由 */
        )}
      </Content>
    </Layout>
  );
};
```

## 四、集成方案

### 4.1 iframe 通信

权限中心与用户中心通过 iframe 嵌入，使用 postMessage 通信：

```typescript
// 权限中心 → 用户中心
const iframeRef = useRef<HTMLIFrameElement>(null);

const sendMessageToUserCenter = (message: any) => {
  iframeRef.current?.contentWindow?.postMessage({
    type: 'PERMISSION_CENTER',
    payload: message,
  }, '*');
};

// 监听用户中心消息
useEffect(() => {
  const handleMessage = (event: MessageEvent) => {
    if (event.data.type === 'USER_CENTER') {
      // 处理用户中心返回的消息
      console.log('Received from user center:', event.data.payload);
    }
  };

  window.addEventListener('message', handleMessage);
  return () => window.removeEventListener('message', handleMessage);
}, []);
```

## 七、风险与应对

| 风险 | 影响 | 应对措施 |
|------|------|---------|
| iframe 跨域问题 | 用户中心无法加载 | 配置 CORS：`Access-Control-Allow-Origin: *` |
| Token 过期不同步 | 用户体验差 | 统一 Token 刷新机制，iframe 监听 Token 变化 |
| 数据迁移失败 | 数据丢失 | 迁移前备份，灰度验证 |
| API 不兼容 | 功能异常 | 保持旧 API 兼容，逐步迁移 |
| 性能问题 | 加载慢 | 优化查询，添加缓存 |
