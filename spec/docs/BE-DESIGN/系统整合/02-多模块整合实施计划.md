## 一、总体规划

### 1.1 实施策略

采用**渐进式集成**策略，确保系统在改造过程中始终可用：

```
阶段一：后端基础能力（1周）
    ↓
阶段二：用户管理模块（1周）
    ↓
阶段三：前端集成（3-5天）
    ↓
阶段四：测试与优化（2-3天）
```

### 1.2 技术栈保留

| 模块 | 技术栈 | 改造程度 | 说明 |
|------|--------|----------|------|
| 权限中心后端 | Spring Boot 3.x | ⬆️ 增强 | 新增用户管理 API |
| 权限中心前端 | React 18 + AntD | ⬆️ 改造 | 新增 Tab 切换、iframe 嵌入 |
| 用户中心前端 | Vue 3 + TDesign | ⬇️ 精简 | 移除登录，对接权限中心 API |
| 用户中心后端 | NestJS | ❌ 废弃 | 迁移到 Spring Boot |

```typescript
import { computed } from 'vue';
import { useUserStore } from '@/stores/modules/user';

export function usePermissions() {
  const userStore = useUserStore();

  const permissions = computed(() =>
    new Set(userStore.permissions || [])
  );

  const hasPermission = (code: string) => {
    if (userStore.isAdmin) return true;
    return permissions.value.has(code);
  };

  return {
    permissions,
    hasPermission,
  };
}
```

## 七、风险应对

### 7.1 技术风险

| 风险 | 影响 | 应对措施 |
|------|------|---------|
| iframe 跨域 | 用户中心无法加载 | 配置 CORS，使用同域名部署 |
| Token 同步 | 体验不一致 | 统一 Token 管理，定期刷新 |
| 性能问题 | 加载慢 | 添加缓存，优化查询 |
