## 2. 数据库设计

### 2.1 项目表结构

```sql
CREATE TABLE IF NOT EXISTS `project` (
    `id`            BIGINT UNSIGNED NOT NULL AUTO_INCREMENT COMMENT '主键ID',
    `code`          VARCHAR(64)     NOT NULL COMMENT '项目编码，全局唯一',
    `name`          VARCHAR(128)    NOT NULL COMMENT '项目名称',
    `description`   VARCHAR(256)    DEFAULT NULL COMMENT '描述',
    `status`        VARCHAR(16)     NOT NULL DEFAULT 'ENABLED' COMMENT '状态：ENABLED/DISABLED',
    `gmt_create`    DATETIME        NOT NULL DEFAULT CURRENT_TIMESTAMP COMMENT '创建时间',
    `gmt_modified`  DATETIME        NOT NULL DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP COMMENT '修改时间',
    `deleted`       TINYINT(1)      NOT NULL DEFAULT 0 COMMENT '逻辑删除：0=未删除，1=已删除',
    PRIMARY KEY (`id`),
    UNIQUE KEY `uk_code` (`code`, `deleted`),
    KEY `idx_status` (`status`)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='项目表';
```

### 2.2 索引说明

| 索引名 | 类型 | 字段 | 说明 |
|--------|------|------|------|
| PRIMARY | 主键 | id | 自增主键 |
| uk_code | 唯一 | (code, deleted) | code 唯一，含 deleted 支持逻辑删除后重建 |
| idx_status | 普通 | status | 按状态筛选 |

## 3. 接口设计

### 3.1 接口列表

| 序号 | 方法 | 路径 | 说明 |
|------|------|------|------|
| 1 | POST | /projects | 新增项目 |
| 2 | PUT | /projects/{id} | 编辑项目 |
| 3 | DELETE | /projects/{id} | 删除项目 |
| 4 | GET | /projects | 查询项目列表（分页） |
| 5 | GET | /projects/all | 查询所有启用项目（下拉框用） |

### 3.2 接口详细设计

#### 3.2.1 新增项目

**路径**：`POST /projects`

**请求体**：`CreateProjectDTO`

| 参数 | 类型 | 必填 | 校验规则 | 说明 |
|------|------|------|----------|------|
| code | String | 是 | `@NotBlank` `@Pattern(^[A-Z][A-Z0-9_]{0,63}$)` | 项目编码 |
| name | String | 是 | `@NotBlank` `@Size(max=128)` | 项目名称 |
| description | String | 否 | `@Size(max=256)` | 描述 |

**响应体**：`ApiResponse<ProjectVO>`

```json
{
  "code": 200,
  "message": "success",
  "data": {
    "id": 1,
    "code": "P1",
    "name": "项目一",
    "description": "测试项目",
    "status": "ENABLED",
    "gmtCreate": "2026-03-15T10:00:00",
    "gmtModified": "2026-03-15T10:00:00"
  }
}
```

**错误响应**：

| 错误码 | 错误信息 | 触发条件 |
|--------|----------|----------|
| 142301 | 项目编码已存在：%s | code 重复 |

#### 3.2.2 编辑项目

**路径**：`PUT /projects/{id}`

**路径参数**：id（Long）

**请求体**：`UpdateProjectDTO`

| 参数 | 类型 | 必填 | 校验规则 | 说明 |
|------|------|------|----------|------|
| name | String | 否 | `@Size(max=128)` | 项目名称 |
| description | String | 否 | `@Size(max=256)` | 描述 |
| status | String | 否 | 枚举校验 | ENABLED/DISABLED |

**注意**：code 字段不在 DTO 中，不允许修改。

**响应体**：`ApiResponse<ProjectVO>`

**错误响应**：

| 错误码 | 错误信息 | 触发条件 |
|--------|----------|----------|
| 142301 | 项目不存在 | id 无效 |

#### 3.2.3 删除项目

**路径**：`DELETE /projects/{id}`

**响应体**：`ApiResponse<Void>`

**错误响应**：

| 错误码 | 错误信息 | 触发条件 |
|--------|----------|----------|
| 142301 | 项目不存在 | id 无效 |
| 142302 | 该项目已被用户角色引用 | user_role 有记录 |
| 142303 | 该项目已被用户权限引用 | user_permission 有记录 |

#### 3.2.4 查询项目列表

**路径**：`GET /projects`

**Query 参数**：`ProjectQueryDTO`

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| code | String | 否 | 模糊匹配 |
| name | String | 否 | 模糊匹配 |
| status | String | 否 | 精确匹配 |
| pageNum | Integer | 否 | 默认 1 |
| pageSize | Integer | 否 | 默认 20 |

**响应体**：`ApiResponse<PageResult<ProjectVO>>`

#### 3.2.5 查询所有启用项目

**路径**：`GET /projects/all`

**响应体**：`ApiResponse<List<ProjectVO>>`

用于下拉框选择，仅返回 ENABLED 状态的项目。

## 5. 业务逻辑设计

### 5.1 新增项目

**方法**：`ProjectManager.createProject(CreateProjectDTO dto)`

**处理步骤**：

1. **参数校验**
   - 输入：CreateProjectDTO
   - 处理：`@Valid` 注解自动校验
   - 异常：400 - 参数校验失败

2. **code 唯一性校验**
   - 输入：dto.code
   - 处理：`ProjectService.getByCode(code)` 查询（含 deleted=0）
   - 异常：142301 - 项目编码已存在

3. **创建 DO 并保存**
   - 处理：Converter 转换，设置 status=ENABLED，调用 `ProjectService.save()`
   - 输出：返回 ProjectVO

### 5.2 编辑项目

**方法**：`ProjectManager.updateProject(Long id, UpdateProjectDTO dto)`

**处理步骤**：

1. **查询现有记录**
   - 处理：`ProjectService.getById(id)`
   - 异常：142301 - 项目不存在

2. **更新字段**
   - 处理：仅更新 DTO 中非 null 的字段
   - 调用 `ProjectService.updateById()`

3. **返回更新后数据**

### 5.3 删除项目

**方法**：`ProjectManager.deleteProject(Long id)`

**处理步骤**：

1. **查询现有记录**
   - 异常：142301 - 项目不存在

2. **用户角色引用检查**
   - 处理：`UserRoleService.countByProjectId(project.code)`
   - 异常：142302 - 该项目已被用户角色引用

3. **用户权限引用检查**
   - 处理：`UserPermissionService.countByProjectId(project.code)`
   - 异常：142303 - 该项目已被用户权限引用

4. **逻辑删除**
   - 处理：`ProjectService.removeById(id)`（MyBatis-Plus 逻辑删除）

### 5.4 查询项目列表

**方法**：`ProjectManager.listProjects(ProjectQueryDTO dto)`

**处理步骤**：

1. 构建查询条件（支持 code、name 模糊匹配，status 精确匹配）
2. 调用 `ProjectService.page()` 分页查询
3. 返回分页结果

### 5.5 查询所有启用项目

**方法**：`ProjectManager.listAllEnabled()`

**处理步骤**：

1. 查询所有 status=ENABLED 的项目
2. 返回列表（不分页）

## 6. 异常处理设计

本模块新增错误码：

| 枚举名 | 错误码 | 错误信息 | 说明 |
|--------|--------|----------|------|
| PROJECT_NOT_FOUND | 142301 | 项目不存在 | 按 id 查询不到 |
| PROJECT_CODE_EXISTS | 142302 | 项目编码已存在：%s | code 唯一性冲突 |
| PROJECT_USED_BY_USER_ROLE | 142303 | 该项目已被用户角色引用 | 删除前置检查 |
| PROJECT_USED_BY_USER_PERMISSION | 142304 | 该项目已被用户权限引用 | 删除前置检查 |
