# 模块系分：角色管理

> 基于 PRD `03-角色管理模块.md`，全局设计参见 `00-全局设计与项目规格.md`

## 1. 模块概述

| 项 | 说明 |
|----|------|
| 模块名称 | 角色管理（Role） |
| 功能范围 | 角色 CRUD、角色-权限点分配（全量覆盖）、删除前置校验 |
| 涉及表 | role, role_permission |
| 对外依赖 | PermissionService（校验权限点存在性） |

## 3. 接口设计

### 3.1 接口列表

| 序号 | 方法 | 路径 | 说明 |
|------|------|------|------|
| 1 | POST | /roles | 新增角色 |
| 2 | PUT | /roles/{id} | 编辑角色 |
| 3 | DELETE | /roles/{id} | 删除角色 |
| 4 | GET | /roles | 查询角色列表（分页） |
| 5 | POST | /roles/{roleId}/permissions | 为角色分配权限（全量覆盖） |
| 6 | GET | /roles/{roleId}/permissions | 查询角色已分配的权限 |

### 3.2 接口详细设计

#### 3.2.1 新增角色

**路径**：`POST /roles`

**请求体**：`CreateRoleDTO`

| 参数 | 类型 | 必填 | 校验规则 | 说明 |
|------|------|------|----------|------|
| code | String | 是 | `@NotBlank` `@Pattern(^[A-Z][A-Z0-9_]{1,63}$)` | 角色编码 |
| name | String | 是 | `@NotBlank` `@Size(max=128)` | 角色名称 |
| roleScope | String | 是 | `@NotBlank` 枚举校验 | GLOBAL/PROJECT |
| roleDomain | String | 是 | `@NotBlank` 枚举校验 | APP/PERM_CENTER |
| description | String | 否 | `@Size(max=256)` | 描述 |

**响应体**：`ApiResponse<RoleVO>`

**错误响应**：

| 错误码 | 错误信息 | 触发条件 |
|--------|----------|----------|
| 142102 | 角色编码已存在 | code 重复 |

#### 3.2.2 编辑角色

**路径**：`PUT /roles/{id}`

**请求体**：`UpdateRoleDTO`

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| name | String | 否 | 角色名称 |
| roleScope | String | 否 | GLOBAL/PROJECT |
| roleDomain | String | 否 | APP/PERM_CENTER |
| status | String | 否 | ENABLED/DISABLED |
| description | String | 否 | 描述 |

**注意**：code 不可修改。

**错误响应**：

| 错误码 | 错误信息 | 触发条件 |
|--------|----------|----------|
| 142101 | 角色不存在 | id 无效 |

#### 3.2.3 删除角色

**路径**：`DELETE /roles/{id}`

**错误响应**：

| 错误码 | 错误信息 | 触发条件 |
|--------|----------|----------|
| 142101 | 角色不存在 | id 无效 |
| 142104 | 该角色已被用户引用 | user_role 有记录 |

**处理**：删除角色时同时清理 role_permission 关联数据。

#### 3.2.4 查询角色列表

**路径**：`GET /roles`

**Query 参数**：`RoleQueryDTO`

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| code | String | 否 | 模糊匹配 |
| name | String | 否 | 模糊匹配 |
| roleScope | String | 否 | 精确匹配 |
| roleDomain | String | 否 | 精确匹配 |
| status | String | 否 | 精确匹配 |
| pageNum | Integer | 否 | 默认 1 |
| pageSize | Integer | 否 | 默认 20 |

**响应体**：`ApiResponse<PageResult<RoleVO>>`

#### 3.2.5 为角色分配权限

**路径**：`POST /roles/{roleId}/permissions`

**请求体**：`AssignRolePermissionDTO`

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| permissionCodes | List\<String\> | 是 | 权限编码列表（全量覆盖） |

**响应体**：`ApiResponse<Void>`

**错误响应**：

| 错误码 | 错误信息 | 触发条件 |
|--------|----------|----------|
| 142101 | 角色不存在 | roleId 无效 |
| 142103 | 角色已禁用 | 角色 DISABLED |
| 142001 | 权限点不存在 | permissionCode 无效 |
| 142008 | 权限点已禁用 | 权限点 DISABLED |

#### 3.2.6 查询角色已分配的权限

**路径**：`GET /roles/{roleId}/permissions`

**响应体**：`ApiResponse<List<String>>`（权限编码列表）

## 5. 业务逻辑设计

### 5.1 新增角色

**方法**：`RoleManager.createRole(CreateRoleDTO dto)`

**处理步骤**：

1. **参数校验** — `@Valid` 自动校验
2. **code 唯一性校验** — `RoleService.getByCode(code)`，异常：142102
3. **枚举校验** — roleScope、roleDomain 枚举值校验
4. **创建 DO** — Converter 转换，status=ENABLED
5. **保存** — `RoleService.save()`
6. **返回 RoleVO**

### 5.2 编辑角色

**方法**：`RoleManager.updateRole(Long id, UpdateRoleDTO dto)`

**处理步骤**：

1. **查询现有记录** — `RoleService.getById(id)`，异常：142101
2. **枚举校验**（若有变更）
3. **更新非 null 字段** — `RoleService.updateById()`
4. **返回 RoleVO**

### 5.3 删除角色

**方法**：`RoleManager.deleteRole(Long id)`

**处理步骤**：

1. **查询现有记录** — 异常：142101
2. **用户引用检查** — `UserRoleService.countByRoleId(id)`，异常：142104
3. **清理角色-权限关联** — `RolePermissionService.deleteByRoleId(id)`
4. **逻辑删除角色** — `RoleService.removeById(id)`

**事务**：步骤 3-4 在同一事务中，`@Transactional` 注解在 Manager 层。

### 5.4 为角色分配权限（全量覆盖）

**方法**：`RoleManager.assignPermissions(Long roleId, AssignRolePermissionDTO dto)`

**处理步骤**：

1. **校验角色** — 存在且 ENABLED，异常：142101 / 142103
2. **校验权限点** — 遍历 permissionCodes，逐个校验存在且 ENABLED
   - `PermissionService.getByCode(code)`
   - 异常：142001 / 142008
3. **全量覆盖** — 删除该角色原有所有权限关联
   - `RolePermissionService.deleteByRoleId(roleId)`
4. **批量插入** — 构建 RolePermissionDO 列表，批量保存
   - `RolePermissionService.saveBatch(list)`

**事务**：步骤 3-4 在同一事务中。

### 5.5 查询角色已分配的权限

**方法**：`RoleManager.getRolePermissions(Long roleId)`

**处理步骤**：

1. **查询关联记录** — `RolePermissionService.listByRoleId(roleId)`
2. **提取 permissionCode 列表** — stream().map().collect()
3. **返回 List\<String\>**

## 7. 前端页面设计

### 7.1 角色管理页面（权限树形勾选）

**页面路径**：`/roles`

**页面布局**：
- 顶部工具栏：新建角色按钮、刷新按钮
- 主体：角色列表表格（支持分页）
- 弹窗：新建/编辑角色表单、权限配置弹窗

**功能点**：
- 角色列表展示（表格）
- 新建角色（Modal 弹窗）
- 编辑角色（Modal 弹窗）
- 删除角色（确认弹窗）
- **权限配置（树形勾选弹窗）**

**树形组件设计**：

```tsx
interface PermissionCheckTreeNode {
  key: string;           // 权限编码
  title: ReactNode;      // 权限名称 + 类型标签
  children?: PermissionCheckTreeNode[];
  disableCheckbox?: boolean;  // 禁用状态
}
```

**功能特性**：
1. **树形展示**：使用 Ant Design Tree 组件，checkbox 模式
2. **父子联动**：勾选父节点自动勾选所有子节点
3. **搜索过滤**：支持按权限编码/名称搜索
4. **展开控制**：展开全部、折叠全部
5. **统计显示**：显示已选择权限数量

**交互逻辑**：
1. 打开弹窗时加载权限树和已选权限
2. 勾选/取消勾选权限节点
3. 点击确定保存配置（全量覆盖）

### 7.3 实现要点

**前端组件**：
```tsx
<Modal title={`配置权限 - ${roleName}`} ...>

    </div>
    <Tree
      checkable
      treeData={treeData}
      checkedKeys={checkedKeys}
      onCheck={handleCheck}
      expandedKeys={expandedKeys}
      onExpand={setExpandedKeys}

  </div>

```

**数据处理**：
1. 从 `/permissions/tree` 获取权限树数据
2. 从 `/roles/{roleCode}/permissions` 获取已选权限
3. 将已选权限设置为 checkedKeys
