# 安全规范

## 1. 输入校验

- **参数校验**：所有外部输入必须进行校验，包括：参数类型、长度、格式、范围等
- **SQL注入防护**：使用参数化查询，禁止拼接SQL语句
- **XSS防护**：对用户输入进行转义处理
- **文件上传**：限制文件类型、大小，对文件名进行校验

## 2. 敏感信息处理

- **密码存储**：使用加密算法（如BCrypt）存储密码，禁止明文存储
- **敏感信息传输**：使用HTTPS传输敏感信息
- **日志脱敏**：禁止在日志中输出密码、身份证号、银行卡号等敏感信息
- **数据库加密**：敏感字段（如手机号、身份证号）考虑加密存储

## 3. 权限控制

- **接口权限**：所有接口必须进行权限校验
- **数据权限**：根据用户角色和数据权限过滤数据
- **操作日志**：记录关键操作的审计日志

## 4. 其他安全规范

- **HTTPS**：生产环境必须使用HTTPS
- **CSRF防护**：使用CSRF Token防护
- **限流**：对接口进行限流，防止恶意请求
- **依赖安全**：定期检查依赖库的安全漏洞
