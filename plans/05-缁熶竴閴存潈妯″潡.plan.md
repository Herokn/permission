## 1. 模块范围

实现统一鉴权能力：单权限鉴权和批量鉴权。鉴权逻辑遵循 `DENY > ALLOW > 角色权限 > 默认拒绝` 优先级。本模块不操作数据库，纯读取操作，依赖其他模块的 Service。

## 3. 技术 Spec

### 3.1 AuthzResult（内部对象）

```java
package com.permission.service.model;

/**
 * 鉴权内部结果对象（Service 层使用，不对外暴露）
 */
public class AuthzResult {
    private boolean allowed;
    private String reason;

    private AuthzResult(boolean allowed, String reason) {
        this.allowed = allowed;
        this.reason = reason;
    }

    public static AuthzResult allowed(String reason) {
        return new AuthzResult(true, reason);
    }

    public static AuthzResult denied(String reason) {
        return new AuthzResult(false, reason);
    }

    // getters
}
```

### 3.2 AuthzService 接口

```java
public interface AuthzService {
    /**
     * 核心鉴权方法
     * @param userId 用户ID
     * @param permissionCode 权限编码
     * @param projectId 项目ID（null 表示全局鉴权）
     * @return AuthzResult 鉴权结果（含 reason）
     */
    AuthzResult check(String userId, String permissionCode, String projectId);
}
```

### 3.3 AuthzServiceImpl 核心算法

```java
@Service
@Slf4j
public class AuthzServiceImpl implements AuthzService {

    @Autowired private PermissionService permissionService;
    @Autowired private UserPermissionService userPermissionService;
    @Autowired private UserRoleService userRoleService;
    @Autowired private RoleService roleService;
    @Autowired private RolePermissionService rolePermissionService;

    @Override
    public AuthzResult check(String userId, String permissionCode, String projectId) {
        // Step 0: 校验权限点
        PermissionDO permission = permissionService.getByCode(permissionCode);
        if (permission == null || !CommonStatusEnum.ENABLED.getCode().equals(permission.getStatus())) {
            return AuthzResult.denied("权限点无效或已禁用");
        }

        // Step 1: 检查用户直接 DENY（优先级最高）
        UserPermissionDO denyRecord = userPermissionService.findDeny(userId, permissionCode, projectId);
        if (denyRecord != null) {
            String projectInfo = denyRecord.getProjectId() != null
                ? "（项目 " + denyRecord.getProjectId() + "）" : "（全局）";
            return AuthzResult.denied("被直接 DENY 覆盖" + projectInfo);
        }

        // Step 2: 检查用户直接 ALLOW
        UserPermissionDO allowRecord = userPermissionService.findAllow(userId, permissionCode, projectId);
        if (allowRecord != null) {
            String projectInfo = allowRecord.getProjectId() != null
                ? "（项目 " + allowRecord.getProjectId() + "）" : "（全局）";
            return AuthzResult.allowed("来自直接授权 ALLOW" + projectInfo);
        }

        // Step 3: 检查用户角色权限
        List<UserRoleDO> userRoles = userRoleService.listByUserIdAndProjectId(userId, projectId);
        for (UserRoleDO userRole : userRoles) {
            RoleDO role = roleService.getById(userRole.getRoleId());
            if (role == null || !CommonStatusEnum.ENABLED.getCode().equals(role.getStatus())) {
                continue;  // 跳过不存在或禁用的角色
            }
            boolean hasPermission = rolePermissionService.exists(role.getId(), permissionCode);
            if (hasPermission) {
                String projectInfo = userRole.getProjectId() != null
                    ? "（项目 " + userRole.getProjectId() + "）" : "（全局）";
                return AuthzResult.allowed("来自角色 " + role.getCode() + " 授权" + projectInfo);
            }
        }

        // Step 4: 默认拒绝
        return AuthzResult.denied("默认拒绝：无匹配授权");
    }
}
```

### 3.4 AuthzManager 接口

```java
public interface AuthzManager {
    AuthzResultVO check(AuthzCheckDTO dto);
    AuthzBatchResultVO checkBatch(AuthzBatchCheckDTO dto);
}
```

### 3.5 AuthzManagerImpl 实现要点

#### check

```
1. 参数校验:
   if userId == blank || permissionCode == blank → 抛 AUTHZ_PARAM_INVALID
2. 调用核心鉴权:
   AuthzResult result = authzService.check(dto.userId, dto.permissionCode, dto.projectId)
3. 构建 AuthzResultVO:
   vo.allowed = result.isAllowed()
   vo.reason = dto.explain == true ? result.getReason() : null
4. 返回 vo
```

#### checkBatch

```
1. 参数校验:
   if userId == blank || permissionCodes == empty → 抛 AUTHZ_PARAM_INVALID
2. 遍历 permissionCodes:
   Map<String, Boolean> results = new LinkedHashMap<>()
   for code in dto.permissionCodes:
     AuthzResult result = authzService.check(dto.userId, code, dto.projectId)
     results.put(code, result.isAllowed())
3. 构建 AuthzBatchResultVO(results)
4. 返回
```

### 3.8 AuthzController

```java
@RestController
@RequestMapping("/authz")
public class AuthzController {

    @PostMapping("/check")
    public ApiResponse<AuthzResultVO> check(@RequestBody @Valid AuthzCheckDTO dto)

    @PostMapping("/check-batch")

```

## 5. 验收标准

- [ ] 单权限鉴权接口正常工作
- [ ] 批量鉴权接口正常工作
- [ ] DENY 优先级最高，覆盖 ALLOW 和角色权限
- [ ] 同时检查 projectId 和 NULL（全局）两类数据
- [ ] 角色 DISABLED 时跳过该角色的权限检查
- [ ] 权限点 DISABLED 时直接返回 DENIED
- [ ] explain=false 时 reason 返回 null
- [ ] explain=true 时 reason 包含来源信息（角色名/项目ID等）
- [ ] 鉴权结果通过 allowed=false 表达拒绝，不抛异常
- [ ] 仅参数为空时才抛 AUTHZ_PARAM_INVALID 异常
