## 2. 执行阶段

### 阶段 1：数据模型定义（DAL 层）

**任务清单**：

- [ ] 1.1 创建 `ProjectDO` 数据对象
  - 路径：`permission-dal/src/main/java/com/permission/dal/dataobject/ProjectDO.java`
  - 字段：id, code, name, description, status, gmtCreate, gmtModified, deleted
  - 注解：`@TableName("project")`

- [ ] 1.2 创建 `ProjectMapper` 接口
  - 路径：`permission-dal/src/main/java/com/permission/dal/mapper/ProjectMapper.java`
  - 继承：`BaseMapper<ProjectDO>`

- [ ] 1.3 创建数据库迁移脚本
  - 路径：`permission-bootstrap/src/main/resources/db/migration/V7__project_table.sql`
  - 内容：创建 project 表

**依赖**：无

---

### 阶段 2：服务层实现（Service 层）

**任务清单**：

- [ ] 2.1 创建 `ProjectService` 接口
  - 路径：`permission-service/src/main/java/com/permission/service/ProjectService.java`
  - 方法：
    - `ProjectDO getByCode(String code)`
    - `ProjectDO getById(Long id)`
    - `void save(ProjectDO project)`
    - `void updateById(ProjectDO project)`
    - `void removeById(Long id)`
    - `Page<ProjectDO> page(Page<ProjectDO> page, LambdaQueryWrapper<ProjectDO> wrapper)`
    - `List<ProjectDO> listByStatus(String status)`

- [ ] 2.2 创建 `ProjectServiceImpl` 实现类
  - 路径：`permission-service/src/main/java/com/permission/service/impl/ProjectServiceImpl.java`
  - 注入：`ProjectMapper`
  - 实现所有接口方法

**依赖**：阶段 1

---

### 阶段 3：业务层实现（Manager 层）

**任务清单**：

- [ ] 3.1 创建 DTO/VO 类
  - 路径：`permission-biz/src/main/java/com/permission/biz/dto/project/`
    - `CreateProjectDTO.java`
    - `UpdateProjectDTO.java`
    - `ProjectQueryDTO.java`
  - 路径：`permission-biz/src/main/java/com/permission/biz/vo/project/`
    - `ProjectVO.java`

- [ ] 3.2 创建 `ProjectWebConverter` 转换器
  - 路径：`permission-biz/src/main/java/com/permission/biz/converter/ProjectWebConverter.java`
  - 使用 MapStruct

- [ ] 3.3 创建 `ProjectManager` 接口
  - 路径：`permission-biz/src/main/java/com/permission/biz/manager/ProjectManager.java`
  - 方法：
    - `ProjectVO createProject(CreateProjectDTO dto)`
    - `ProjectVO updateProject(Long id, UpdateProjectDTO dto)`
    - `void deleteProject(Long id)`
    - `PageResult<ProjectVO> listProjects(ProjectQueryDTO dto)`
    - `List<ProjectVO> listAllEnabled()`

- [ ] 3.4 创建 `ProjectManagerImpl` 实现类
  - 路径：`permission-biz/src/main/java/com/permission/biz/manager/impl/ProjectManagerImpl.java`
  - 注入：`ProjectService`, `UserRoleService`, `UserPermissionService`
  - 实现业务逻辑：
    - code 唯一性校验
    - 删除前置检查（用户角色、用户权限引用）
    - 分页查询

**依赖**：阶段 2

---

### 阶段 4：错误码定义

**任务清单**：

- [ ] 4.1 更新 `ErrorCode` 枚举
  - 路径：`permission-common/src/main/java/com/permission/common/exception/ErrorCode.java`
  - 新增：

    - `PROJECT_USED_BY_USER_PERMISSION("142304", "该项目已被用户权限引用")`

**依赖**：无

---

### 阶段 5：Web 控制器实现

**任务清单**：

- [ ] 5.1 创建 `ProjectController`
  - 路径：`permission-web/src/main/java/com/permission/web/controller/ProjectController.java`
  - 接口：
    - `POST /projects` - 新增项目
    - `PUT /projects/{id}` - 编辑项目
    - `DELETE /projects/{id}` - 删除项目

**依赖**：阶段 3

## 5. 风险点

1. **删除项目时的引用检查**：需要确保 UserRoleService 和 UserPermissionService 提供按 projectId 统计的方法
2. **前端路由和菜单**：需要与现有布局兼容
3. **项目选择组件**：需要考虑空项目列表的情况
