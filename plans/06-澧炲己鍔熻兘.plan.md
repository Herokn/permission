### 3.3 审计日志

#### audit_log 表

```sql
CREATE TABLE audit_log (
    id          BIGINT AUTO_INCREMENT PRIMARY KEY,
    operator    VARCHAR(64)   NOT NULL COMMENT '操作人',
    module      VARCHAR(32)   NOT NULL COMMENT '模块(PERMISSION/ROLE/USER_AUTH)',
    action      VARCHAR(32)   NOT NULL COMMENT '操作(CREATE/UPDATE/DELETE/ASSIGN/REVOKE)',
    target_type VARCHAR(32)   NOT NULL COMMENT '目标类型',
    target_id   VARCHAR(128)  NOT NULL COMMENT '目标ID',
    detail      TEXT          COMMENT '操作详情(JSON)',
    ip_address  VARCHAR(45)   COMMENT '操作人IP',
    gmt_create  DATETIME      NOT NULL DEFAULT CURRENT_TIMESTAMP,
    INDEX idx_operator (operator),
    INDEX idx_module (module),
    INDEX idx_gmt_create (gmt_create)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='操作审计日志';
```

#### @AuditLog 注解

```java
@Target(ElementType.METHOD)
@Retention(RetentionPolicy.RUNTIME)
public @interface AuditLog {
    String module();       // PERMISSION / ROLE / USER_AUTH
    String action();       // CREATE / UPDATE / DELETE / ASSIGN / REVOKE
    String targetType();   // permission / role / user_role / user_permission
    String description() default "";
}
```

#### AuditLogAspect

```java
@Aspect
@Component
@Slf4j
public class AuditLogAspect {
    @Autowired private AuditLogMapper auditLogMapper;
    @Autowired private HttpServletRequest request;

    @Around("@annotation(auditLog)")
    public Object around(ProceedingJoinPoint pjp, AuditLog auditLog) throws Throwable {
        Object result = pjp.proceed();
        try {
            AuditLogDO logDO = new AuditLogDO();
            logDO.setOperator(extractOperator(request));  // 从 header X-User-Id 获取
            logDO.setModule(auditLog.module());
            logDO.setAction(auditLog.action());
            logDO.setTargetType(auditLog.targetType());
            logDO.setTargetId(extractTargetId(pjp));
            logDO.setDetail(buildDetail(pjp));
            logDO.setIpAddress(getClientIp(request));
            auditLogMapper.insert(logDO);
        } catch (Exception e) {
            log.warn("审计日志记录失败", e);  // 审计失败不影响业务
        }
        return result;
    }
}
```

使用方式（在 Manager 层标注）：
```java
@AuditLog(module = "PERMISSION", action = "CREATE", targetType = "permission")
public PermissionVO createPermission(CreatePermissionDTO dto) { ... }
```

### 3.4 鉴权缓存（Caffeine）

```java
public interface AuthzCacheService {
    /** 带缓存的鉴权 */
    AuthzResult checkWithCache(String userId, String permissionCode, String projectId);
    /** 清除指定用户的所有缓存 */
    void evictUser(String userId);
    /** 清除与指定权限编码相关的所有缓存 */
    void evictPermission(String permissionCode);
    /** 清除所有缓存 */
    void evictAll();
}
```

实现：
```java
@Service
public class AuthzCacheServiceImpl implements AuthzCacheService {
    private final Cache<String, AuthzResult> cache = Caffeine.newBuilder()
        .maximumSize(10_000)
        .expireAfterWrite(Duration.ofMinutes(5))
        .recordStats()
        .build();

    @Autowired private AuthzService authzService;

    @Override
    public AuthzResult checkWithCache(String userId, String permissionCode, String projectId) {
        String key = buildKey(userId, permissionCode, projectId);
        return cache.get(key, k -> authzService.check(userId, permissionCode, projectId));
    }

    @Override
    public void evictUser(String userId) {
        cache.asMap().keySet().removeIf(k -> k.startsWith(userId + ":"));
    }

    @Override
    public void evictPermission(String permissionCode) {
        cache.asMap().keySet().removeIf(k -> k.contains(":" + permissionCode + ":"));
    }

    @Override
    public void evictAll() { cache.invalidateAll(); }

    private String buildKey(String userId, String permissionCode, String projectId) {
        return userId + ":" + permissionCode + ":" + (projectId == null ? "_GLOBAL_" : projectId);
    }
}
```

### 3.5 缓存失效触发点

在以下 Manager 方法中调用缓存清除：

| 操作 | 清除策略 |
|------|----------|
| 创建/更新/删除权限点 | `evictPermission(code)` |
| 角色分配权限 | `evictAll()`（影响所有拥有该角色的用户） |
| 用户-角色分配/移除 | `evictUser(userId)` |
| 用户直接权限变更 | `evictUser(userId)` |

### 3.6 操作幂等性

在 Service 层实现，关键点：
- **用户-角色分配**：先查 `existsByUserIdAndRoleIdAndProjectId`，存在则直接返回（不报错）
- **用户-角色移除**：查不到记录时直接返回成功
- **用户直接权限授予/排除**：先查是否存在相同记录，存在则更新 effect，不重复插入
- **用户直接权限移除**：查不到记录时直接返回成功
