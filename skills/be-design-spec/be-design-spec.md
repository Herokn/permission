# 后端系分设计规范 Skill

本 Skill 用于指导 AI 编写符合规范的后端系统分析文档（系分文档）。

## 核心职责

1. 将 PRD 需求转换为技术实现方案
2. 确保系分文档包含足够的技术细节供代码生成
3. 遵循项目既定的编码规范和设计规范

## 文档结构要求

### 必需章节

| 章节 | 内容要点 |
|------|----------|
| 1. 模块概述 | 模块基本信息、功能概述、技术栈 |
| 2. 数据库设计 | 表结构、索引、约束、表关系 |
| 3. 接口设计 | API 端点、请求/响应、校验规则、错误码 |
| 4. 代码结构设计 | 分层架构、包结构、类清单 |
| 5. 业务逻辑设计 | 核心流程、校验规则、权限控制 |
| 6. 异常处理设计 | 异常码定义、异常处理策略 |
| 7. 开发检查清单 | 代码规范、功能实现、测试检查 |

### 可选章节

- 日志设计（有特殊日志需求时）
- 性能设计（有性能要求时）
- 安全设计（有安全要求时）
- 单元测试设计（需要详细测试用例时）

## PRD 到系分转换规则

### 数据模型 → 数据库设计

```
PRD 字段定义                    →  补充技术细节
─────────────────────────────────────────────────
字段名、类型、必填、说明    →  数据库类型、长度、默认值、索引、约束
```

**字段类型映射**：
| PRD 类型 | 数据库类型 |
|----------|------------|
| String | VARCHAR(n) |
| Long/Integer | BIGINT/INT |
| Boolean | TINYINT(1) |
| DateTime | DATETIME |
| Enum | VARCHAR 或 ENUM |

### 接口需求 → 接口设计

```
PRD 接口定义                    →  补充技术细节
─────────────────────────────────────────────────
路径、参数、返回、校验      →  HTTP方法、请求结构、响应结构、错误码、权限
```

**RESTful 规范**：
- GET：查询
- POST：创建
- PUT：更新
- DELETE：删除

### 核心逻辑 → 业务逻辑设计

```
PRD 业务规则                    →  补充技术细节
─────────────────────────────────────────────────
触发条件、处理逻辑          →  输入/处理/输出/异常、事务管理、代码实现步骤
```

## 设计规范要点

### 数据库设计

1. **主键**：BIGINT 自增
2. **逻辑删除**：deleted 字段（TINYINT, 0=未删除, 1=已删除）
3. **时间字段**：created_at, updated_at（DATETIME）
4. **索引原则**：
   - 主键必须有主键索引
   - 唯一字段必须有唯一索引
   - 查询条件字段建立普通索引

### 接口设计

1. **统一响应格式**：
```json
{
  "code": 200,
  "message": "操作成功",
  "data": {}
}
```

2. **错误码格式**：6位编码 `XXYYZZ`
   - XX：模块编码
   - YY：子模块编码
   - ZZ：具体错误序号

3. **参数校验**：使用 jakarta.validation 注解

### 分层架构

```
├── api层          # 对外接口定义
├── biz-shared层   # 业务共享逻辑接口
├── biz-service-impl层  # 业务实现
├── core层         # 核心领域模型
├── integration层  # 外部集成
└── dal层          # 数据访问层
```

### 命名规范

| 类型 | 命名规范 | 示例 |
|------|----------|------|
| Controller | [Module]Controller | UserController |
| Manager | [Module]Manager | UserManager |
| Service | [Module]Service | UserService |
| DAO | [Module]DAO | UserDAO |
| DO | [Module]DO | UserDO |
| DTO | [Verb][Module]DTO | CreateUserDTO |
| VO | [Module]VO | UserVO |

## 质量检查清单

### 完整性
- [ ] 包含所有必需章节
- [ ] 数据库设计完整（表结构、索引、约束）
- [ ] 接口设计完整（请求、响应、错误码）
- [ ] 业务逻辑完整（流程、规则、异常）

### 准确性
- [ ] 数据库字段类型正确
- [ ] 接口路径和方法正确
- [ ] 业务逻辑符合 PRD
- [ ] 异常码符合规范

### 可执行性
- [ ] AI 可根据文档生成代码
- [ ] 技术细节足够详细
- [ ] 设计决策明确

## 使用场景

1. **新模块开发**：基于 PRD 生成完整系分文档
2. **功能扩展**：基于现有系分文档增量更新
3. **代码评审**：对照系分文档检查实现一致性
